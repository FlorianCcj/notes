Db - Postgres - User Permissions¶

There are no users in Postgres, just role

in the prompt if you have => instead of =# it means that you don t have superuser right

  • psql postgres: login with username postgres

  • psql postgres -U testing: login with another user

  • \du: list roles

  • \q: quit

  • CREATE ROLE <role>;: create a role but without right to login

  • CREATE ROLE <role> WITH LOGIN;: create a role but with right to login

  • CREATE ROLE <role> WITH LOGIN PASSWORD <pwd>;: create a role but with right to login and a password

  • CREATE USER <role> PASSWORD <pwd>;: same as CREATE ROLE <role> WITH LOGIN PASSWORD <pwd>;

  • DROP ROLE <role>: remove a role

  • ALTER ROLE <role> WITH LOGIN;: edit a role to add attribute

Built-in role attributes

  • LOGIN / NOLOGIN: allow (or not) to login to Postgres

  • SUPERUSER / NOSUPERUSER: allow (or not) superuser permissions. A database superuser will bypass other permission checks, except for LOGIN (it must be granted separately).

  • CREATEDB / NOCREATEDB: allow (or not) the ability to create new databases

  • CREATEROLE / NOCREATEROLE: allow (or not) the ability to create new roles

  • CREATEUSER / NOCREATEUSER: allow (or not) the ability to create new users

  • INHERIT / NOINHERIT: allow (or not) the ability to make the privileges inheritable

  • REPLICATION / NOREPLICATION: grant (or not) replication permissions (an advanced topic we’ll not cover)

Group roles

  • CREATE ROLE <groupname>;: create group

  • GRANT <groupname> TO <role>: add a role in a group

  • REVOKE <groupname> FROM <username>: remove role from group

Group role attributes

  • CREATE ROLE employee WITH CREATEDB INHERIT;: every one in employee group will have CREATEDB right
